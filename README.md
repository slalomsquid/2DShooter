UNIX Pholosophy:
Do One Thing Well: Each program should handle a single, specific task and excel at it.

Composability: Design programs to connect, where the output of one program becomes the input for another (piping).

Textual Interfaces: Use plain text streams as a universal interface, avoiding complex binary formats.

Prototypes: Favor building prototypes quickly and iterating, rather than waiting to build the perfect system.

Portability over Efficiency: Prioritize making software run on different machines rather than optimizing for speed at the cost of portability.

Avoid Captive User Interfaces: Programs should not restrict user input or force a specific, interactive flow, allow command arguments or alternative inputs.
